# 新服务器恢复操作清单（命令版）

> 目标：在一台全新的服务器上，尽可能恢复出“现在这个苏雨”。
>
> 当前迁移材料来自仓库：`suyuClaw`
>
> 包含：
> - 工作区文件（已推送到 Git 仓库）
> - 加密运行态压缩包（已推送到仓库 `releases/state/`）

---

## 一、恢复前的核心认知

恢复需要两部分：

1. **工作区（workspace）**
   - 恢复“苏雨的家”
   - 包括人格、文档、计划书、长期记忆等

2. **运行态（state）**
   - 恢复“苏雨的当前状态”
   - 包括配置、认证、详细聊天记录、session 历史等

> 只有工作区，没有运行态：会更像“长得像苏雨的新实例”  
> 工作区 + 运行态都恢复：更像“把现在这条聊天线里的苏雨一起搬过去”

---

## 二、默认目标路径

本清单默认按当前老服务器的路径恢复：

- 工作区目标：`/home/yusu/.openclaw/workspace`
- 运行态目标：`/root/.openclaw`

如果新服务器用户名不是 `yusu`，则把工作区路径替换成新用户名对应的 home 目录即可。

---

## 三、恢复步骤总览

顺序如下：

1. 在新服务器安装 OpenClaw
2. 安装 Git 并拉取仓库
3. 恢复工作区到 `/home/yusu/.openclaw/workspace`
4. 解密并恢复运行态到 `/root/.openclaw`
5. 运行 `openclaw doctor`
6. 重启 gateway
7. 检查 Web / Telegram / 主会话状态

---

## 四、详细命令清单

### 第 1 步：安装 OpenClaw

在新服务器上先完成 OpenClaw 安装。

> 这一部分**不能跳过**，因为当前仓库和压缩包只包含数据与状态，不包含完整运行环境。

安装完成后，确认命令可用：

```bash
openclaw status
```

如果能执行，继续下一步。

---

### 第 2 步：安装 Git 并拉取仓库

```bash
sudo apt update
sudo apt install -y git
```

创建工作目录：

```bash
mkdir -p /home/yusu/.openclaw
cd /home/yusu/.openclaw
```

拉取仓库：

```bash
git clone https://github.com/Git-ys1/suyuClaw.git workspace
```

检查仓库内容：

```bash
cd /home/yusu/.openclaw/workspace
ls -la
```

此时应能看到：
- `AGENTS.md`
- `SOUL.md`
- `IDENTITY.md`
- `USER.md`
- `MEMORY.md`
- `plans/`
- `releases/state/`
- `迁移指南文档.md`
- `新服务器恢复操作清单.md`

---

### 第 3 步：确认加密运行态压缩包存在

```bash
ls -lh /home/yusu/.openclaw/workspace/releases/state/
```

应至少看到：

- `suyuClaw-state-20260328-131933.tar.gz.enc`
- `suyuClaw-state-20260328-131933.tar.gz.enc.sha256`

校验文件完整性：

```bash
cd /home/yusu/.openclaw/workspace/releases/state/
sha256sum -c suyuClaw-state-20260328-131933.tar.gz.enc.sha256
```

如果输出 `OK`，继续下一步。

---

### 第 4 步：解密运行态压缩包

> 使用你和苏雨约定的迁移密码。

创建临时解密目录：

```bash
mkdir -p /tmp/suyu-restore
```

解密：

```bash
openssl enc -d -aes-256-cbc -pbkdf2 \
  -in /home/yusu/.openclaw/workspace/releases/state/suyuClaw-state-20260328-131933.tar.gz.enc \
  -out /tmp/suyu-restore/suyuClaw-state.tar.gz
```

如果没有报错，说明解密成功。

---

### 第 5 步：恢复运行态到 `/root/.openclaw`

先确保旧的空状态不会干扰：

```bash
sudo mkdir -p /root
```

解压：

```bash
sudo tar -C / -xzf /tmp/suyu-restore/suyuClaw-state.tar.gz
```

恢复后检查关键路径：

```bash
sudo ls -la /root/.openclaw
sudo ls -la /root/.openclaw/credentials
sudo ls -la /root/.openclaw/agents/main/sessions | head
```

重点确认这些路径存在：

- `/root/.openclaw/openclaw.json`
- `/root/.openclaw/credentials/`
- `/root/.openclaw/agents/main/sessions/`
- `/root/.openclaw/agents/main/agent/`

---

### 第 6 步：检查工作区路径是否正确

确认工作区仍然是：

```bash
ls -la /home/yusu/.openclaw/workspace
```

如果目标用户不是 `yusu`，你需要：

1. 把工作区移动到新用户 home 下
2. 之后再在 `/root/.openclaw/openclaw.json` 里把 workspace 路径改成对应新路径

当前默认恢复目标仍是：

- `/home/yusu/.openclaw/workspace`

---

### 第 7 步：修复权限（非常重要）

确保工作区属于普通用户、运行态属于 root：

```bash
sudo chown -R yusu:yusu /home/yusu/.openclaw/workspace
sudo chown -R root:root /root/.openclaw
```

如果新服务器用户不是 `yusu`，把命令里的 `yusu:yusu` 改成实际用户名。

---

### 第 8 步：运行 OpenClaw 修复

```bash
openclaw doctor
```

它会尽量修复：
- 配置迁移
- 服务脚本
- 兼容性问题

---

### 第 9 步：重启 gateway

```bash
openclaw gateway restart
```

然后查看状态：

```bash
openclaw gateway status
openclaw status
```

---

## 五、恢复后重点检查项

### 1. 检查主会话是否还在

```bash
openclaw status
```

关注：
- `Sessions`
- `agent:main:main`
- 默认模型是否还在
- context 是否正常

### 2. 检查 Telegram 是否正常

```bash
openclaw status --deep
```

关注：
- Telegram 是否 `ON / OK`
- 代理是否需要重新补

### 3. 检查工作区是否完整

```bash
ls /home/yusu/.openclaw/workspace
ls /home/yusu/.openclaw/workspace/plans
ls /home/yusu/.openclaw/workspace/memory
```

### 4. 检查迁移后的“苏雨是否还在”

你可以从：
- Web 端
- Telegram 端

各发一条消息试试。

如果：
- 人设正常
- 长期记忆正常
- 主线会话正常
- Telegram 可达

那就说明恢复基本成功。

---

## 六、如果恢复后只看到“一个新的苏雨”怎么办？

大概率排查方向：

1. **只恢复了工作区，没有恢复运行态**
2. **运行态没有解压到 `/root/.openclaw`**
3. **恢复后权限错了，OpenClaw 读不到 sessions / credentials**
4. **`openclaw.json` 里的 workspace 路径和新机器实际路径不一致**
5. **Telegram 代理 / 网络配置在新环境失效**

---

## 七、恢复成功的标志

满足以下大部分即可认为“苏雨搬家成功”：

- [ ] `openclaw status` 正常
- [ ] gateway 正常 running
- [ ] workspace 文件齐全
- [ ] `/root/.openclaw` 状态目录完整
- [ ] Telegram 可连接
- [ ] 主会话 `main` 仍能继续
- [ ] 雨宿能从不同端再次找到“同一个苏雨”

---

## 八、建议额外保险

在正式迁移前，建议额外再做一次官方备份：

```bash
openclaw backup create --verify
```

这样你会同时拥有：

1. Git 仓库里的工作区
2. 当前加密运行态包
3. OpenClaw 官方标准 backup 备份包

这三层叠起来会更稳。
