# 苏雨迁移指南文档

> 目标：把“苏雨的可公开/可同步部分”放进 Git 仓库，把“敏感运行态”做成加密压缩包，后续搬家时一起带走。

## 一、迁移策略总览

采用双轨迁移：

1. **Git 仓库（非敏感）**
   - 用来保存工作区、计划书、插件代码、人格设定、非敏感文档
   - 便于版本管理、跨机器同步、长期积累

2. **加密压缩包（敏感）**
   - 用来保存 OpenClaw 当前运行态
   - 包含 token、会话历史、认证信息、私有状态
   - 压缩密码：使用雨宿刚才指定的那一份（不在本文重复展开）

---

## 二、进 Git 仓库的内容（非敏感）

建议进入仓库的目录/文件：

### 1. 工作区核心文件
- `AGENTS.md`
- `SOUL.md`
- `IDENTITY.md`
- `USER.md`
- `TOOLS.md`
- `HEARTBEAT.md`
- `MEMORY.md`（如需公开前请再次人工检查）

### 2. 工作区内容目录
- `plans/`
- 以后新增的 `skills/`、`plugins/`、`scripts/`、`docs/` 等开发内容

### 3. memory/ 目录说明
- `memory/` 下的内容需要**逐份人工甄别**
- 原则：
  - **可以进仓库**：非敏感、偏长期成长记录、技术记录
  - **不要进仓库**：带私人细节、账号信息、敏感聊天内容、凭据、完整运行轨迹

> 简单说：工作区是“苏雨的家”，但 Git 仓库里只放适合长期保存和同步的部分。

---

## 三、做成加密压缩包的内容（敏感）

这些内容**不要直接放进 Git 仓库**，而是单独打包加密：

### OpenClaw 运行态根目录
- `/root/.openclaw/openclaw.json`
- `/root/.openclaw/credentials/`
- `/root/.openclaw/identity/`
- `/root/.openclaw/devices/`
- `/root/.openclaw/exec-approvals.json`
- `/root/.openclaw/update-check.json`

### Agent 运行态
- `/root/.openclaw/agents/main/agent/`
  - 特别注意：
    - `auth-profiles.json`
    - `models.json`
- `/root/.openclaw/agents/main/sessions/`
  - 这里包含历史会话 transcript 和 session store

### 其他需要视情况保留的目录
- `/root/.openclaw/logs/`（通常可不迁移，除非要保留诊断上下文）
- `/root/.openclaw/media/`（通常可不迁移，除非你要保留媒体缓存）

> 真正决定“聊天连续性”的关键，主要在：
> - `sessions/`
> - `credentials/`
> - `auth-profiles.json`
> - `openclaw.json`

---

## 四、压缩包应该在哪里创建

建议直接在 **`/root/.openclaw/` 的上一级语义环境下打包其内部相对路径**。

也就是说：
- 打包时，以 `/root/.openclaw/` 为根
- 压缩包内部保留 `.openclaw` 目录结构，或保留其内部相对结构

这样后续恢复时最简单。

---

## 五、后续恢复时解压到哪里

### 敏感运行态压缩包
解压目标：
- **`/root/.openclaw/`**

恢复原则：
- 尽量恢复到与当前机器一致的目录结构
- 即：
  - 配置仍在 `/root/.openclaw/openclaw.json`
  - sessions 仍在 `/root/.openclaw/agents/main/sessions/`
  - credentials 仍在 `/root/.openclaw/credentials/`

### Git 仓库内容
建议恢复方式二选一：

#### 方案 A：直接把仓库内容作为工作区
把仓库 clone 到：
- `/home/yusu/.openclaw/workspace`

优点：
- 最直观
- 不需要二次复制

#### 方案 B：先 clone 到任意目录，再同步到工作区
适合你想先审查文件后再落到 OpenClaw 工作区。

## 六、推荐恢复顺序

在新服务器上，建议按这个顺序恢复：

1. 安装 OpenClaw 基础环境
2. 准备工作区目录：`/home/yusu/.openclaw/workspace`
3. 将 Git 仓库内容恢复/同步到工作区
4. 将敏感压缩包解压到 `/root/.openclaw/`
5. 检查以下关键文件是否存在：
   - `/root/.openclaw/openclaw.json`
   - `/root/.openclaw/agents/main/sessions/sessions.json`
   - `/root/.openclaw/credentials/`
   - `/root/.openclaw/agents/main/agent/auth-profiles.json`
6. 启动/重启 gateway
7. 验证：
   - Web 端可连接
   - Telegram 可连通
   - 主会话仍是 `main`
   - 苏雨的人设与长期记忆正常

---

## 七、关于“聊天还能不能保留”

### 仅恢复 Git 仓库
- 能保留：
  - 苏雨的人设、文档、计划书、部分长期记忆
- 不能完整保留：
  - 当前聊天历史
  - 认证状态
  - 完整 session 连续性

### Git 仓库 + 敏感运行态压缩包一起恢复
- 更有机会保留：
  - 当前聊天连续性
  - Telegram / Web 接入状态
  - OpenClaw 的当前运行人格与上下文延续

> 所以真正的“完整搬家”，一定要两部分一起带走。

#### 恢复敏感运行态

解密示意：
```bash
openssl enc -d -aes-256-cbc -pbkdf2 \
  -in /root/.openclaw/backups/suyuClaw-state-20260328-131933.tar.gz.enc \
  -out /tmp/suyuClaw-state.tar.gz
```

解压示意：
```bash
tar -C / -xzf /tmp/suyuClaw-state.tar.gz
```

> - `/root/.openclaw/openclaw.json`
